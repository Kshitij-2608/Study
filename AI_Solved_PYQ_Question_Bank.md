## Q1. AO* Algorithm: Solved Example

**Question.** Apply AO* on the following AND-OR graph. Find the optimal solution graph.

```text
S is an OR node.
S -> A cost 1
S -> B cost 3

A is an AND node.
A -> C cost 2
A -> D cost 2

B is an OR node.
B -> E cost 4
B -> F cost 2

Heuristic values:
C=3, D=1, E=2, F=4
```

### Stepwise Solution

**Step 1: Understand AND and OR meaning.**

- At an **OR node**, choose one best child.
- At an **AND node**, all child subproblems must be solved.

**Step 2: Evaluate node A.**

Since A is an AND node, both C and D are required:

$$
Cost(A) = (2 + h(C)) + (2 + h(D))
$$

$$
Cost(A) = (2+3) + (2+1) = 5 + 3 = 8
$$

**Step 3: Evaluate node B.**

Since B is an OR node, choose the cheaper child:

$$
B \to E = 4 + h(E) = 4+2 = 6
$$

$$
B \to F = 2 + h(F) = 2+4 = 6
$$

Both choices have equal cost. Choose either; by left-to-right tie breaking choose E.

$$
Cost(B)=6
$$

**Step 4: Evaluate root S.**

S is an OR node:

$$
S \to A = 1 + Cost(A) = 1+8 = 9
$$

$$
S \to B = 3 + Cost(B) = 3+6 = 9
$$

Both have equal cost. With left-to-right tie breaking, select A. If no tie rule is given, both are optimal.

### Final Exam Answer

AO* is used on AND-OR graphs where OR nodes represent alternatives and AND nodes represent decomposition into required subproblems. For the given graph, \(Cost(A)=8\), \(Cost(B)=6\), and both \(S \to A\) and \(S \to B\) give total cost \(9\). Hence the optimal solution cost is **9**. With left-to-right tie breaking, the solution graph is:

```text
S -> A
A -> C and D
```

## Q2. Hill Climbing PYQ 1: Numeric Function

**Question.** Consider a state space where each state is represented by an integer value. The initial state is \(x=0\), neighboring states are generated by adding 1, and the value of each state is:

$$
f(x)=-(x-3)^2
$$

The goal is to maximize \(f(x)\). What will be the next state selected by hill climbing and why?

### Stepwise Solution

**Step 1: Write current state value.**

Initial state:

$$
x=0
$$

$$
f(0)=-(0-3)^2=-9
$$

**Step 2: Generate neighbor.**

Neighbor is generated by adding 1:

$$
x=1
$$

$$
f(1)=-(1-3)^2=-4
$$

**Step 3: Compare values.**

Hill climbing selects the state with better heuristic/objective value. Since the problem is maximization:

$$
-4 > -9
$$

So \(x=1\) is better than \(x=0\).

### Final Exam Answer

The next state selected by hill climbing is **\(x=1\)**. This is because \(f(0)=-9\) and \(f(1)=-4\), and since the algorithm maximizes \(f(x)\), it moves to the neighboring state with the higher value. Thus hill climbing moves from \(0\) to \(1\).

## Q3. Hill Climbing PYQ 2: Tree with \(h(n)=8-n\)

**Question.** Consider the tree:

```text
0 -> 1, 2
1 -> 3, 4
2 -> 5, 6
5 -> 7, 8
```

Every edge cost is ignored for hill climbing. The heuristic is:

$$
h(n)=8-n
$$

Assume the root node 0 is already expanded. List the next nodes expanded using hill climbing.

### Stepwise Solution

**Step 1: Calculate heuristic of children of 0.**

Children of 0 are 1 and 2.

$$
h(1)=8-1=7
$$

$$
h(2)=8-2=6
$$

Since hill climbing minimizes \(h(n)\), choose node 2.

**Step 2: Expand node 2.**

Children of 2 are 5 and 6.

$$
h(5)=8-5=3
$$

$$
h(6)=8-6=2
$$

Choose node 6 because \(h(6)\) is smaller.

**Step 3: Check next move.**

Node 6 is a leaf in the given tree. Therefore hill climbing stops.

### Final Exam Answer

After root 0 is expanded, hill climbing selects node **2** because \(h(2)=6\) is less than \(h(1)=7\). Then it selects node **6** because \(h(6)=2\) is less than \(h(5)=3\). Since node 6 has no further child, the search stops. The expansion sequence after root is:

```text
2, 6
```

## Q4. BFS PYQ Pattern: State Graph A to K

**Question.** Given the following transition table, start state is A and goal state is K. Apply BFS and draw/write the explored search tree path.

```text
A -> B, C

D -> F

F -> none
G -> H, K
H -> L
I -> none
J -> K
K -> none
L -> none
```

### Stepwise Solution

**Step 1: BFS expands level by level.**

BFS uses a queue and expands the shallowest node first.

**Step 2: Expand levels.**

Level 0:

```text
A
```

Level 1 from A:

```text
B, C
```

Level 2:

```text
From B: E
From C: D, G
```

So level 2 is:

```text
E, D, G
```

Level 3:

```text
From E: I, J
From D: F
From G: H, K
```

The goal K appears at level 3.

**Step 3: Reconstruct parent path.**

K was generated from G, G from C, and C from A.

### Final Exam Answer

BFS expansion order until K is found:

```text
A, B, C, E, D, G, I, J, F, H, K
```

The shortest BFS path is:

$$
A \to C \to G \to K
$$

Since all moves have unit cost, the path cost is:

$$
3

A* uses:

$$
f(n)=g(n)+h(n)

CLOSED = {A, B, E, J}; OPEN = {K(4), C(6), I(11)}

K is the goal.

A* expansion order:

```text
A, B, E, J, K
```

Path returned:

## Q6. Best First Search: Maze/Graph Style

**Question.** Apply Greedy Best First Search from S to G. Use the heuristic values only.

```text
Edges:
S -> A, B
A -> C, D
B -> E
C -> G
D -> G
E -> G

Heuristics:
S:7, A:4, B:2, C:3, D:1, E:5, G:0
```

### Stepwise Solution

**Step 1: Rule for Greedy Best First Search.**

Greedy Best First Search expands the node with the smallest \(h(n)\).

**Step 2: Start at S.**

Expand S. Generate:

```text
A(h=4), B(h=2)
```

Choose B because \(h(B)=2\) is smaller.

**Step 3: Expand B.**

Generate:

```text
E(h=5)
```

OPEN now contains:

```text
A(h=4), E(h=5)
```

Choose A.

**Step 4: Expand A.**

Generate:

```text
C(h=3), D(h=1)
```

OPEN:

```text
D(h=1), C(h=3), E(h=5)
```

Choose D.

**Step 5: Expand D.**

D generates G:

```text
G(h=0)
```

Choose G and stop.

### Final Exam Answer

Expansion order:

```text
S, B, A, D, G
```

Path found:

$$
S \to A \to D \to G
$$

Greedy Best First Search uses only \(h(n)\), so it is fast but not guaranteed to be optimal.

## Q7. CSP PYQ: Circular Seating Arrangement

**Question.** Six guests \(A,B,C,D,E,F\) sit around a round table with chairs numbered 1 to 6. A is fixed in chair 1. Constraints:

- A cannot sit next to B.
- B cannot sit next to C.
- C cannot sit next to D or E.
- F cannot sit next to A or B.

Use backtracking search. Explore names alphabetically and prefer lower-numbered seats.

### Stepwise Solution

**Step 1: Fix A.**

```text
A = 1
```

In a circle, chair 1 is adjacent to chairs 2 and 6.

**Step 2: Assign B.**

B cannot sit next to A, so B cannot be 2 or 6.

Lowest possible seat:

```text
B = 3
```

**Step 3: Assign C.**

C cannot sit next to B, so C cannot be 2 or 4.

Try lower valid seat 5 first:

```text
C = 5
```

But then D and E cannot sit next to C, so seats 4 and 6 become difficult, and F also cannot sit next to A or B. This leads to conflict. Backtrack.

Next try:

```text
C = 6
```

C is not next to B because B is at 3.

**Step 4: Assign D.**

D cannot sit next to C. Since C is at 6, D cannot be 5 or 1. Lowest available valid seat:

```text
D = 2
```

**Step 5: Assign E.**

E cannot sit next to C, so E cannot be 5 or 1. Available seats are 4 and 5. Seat 4 is valid:

```text
E = 4
```

**Step 6: Assign F.**

Remaining seat:

```text
F = 5
```

F is adjacent to E and C, not A or B, so valid.

### Final Exam Answer

One valid seating arrangement is:

| Chair | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---|---|---|---|---|---|
| Guest | A | D | B | E | F | C |

All constraints are satisfied:

- A is not next to B.
- B is not next to C.
- C is not next to D or E.
- F is not next to A or B.

## Q8. CSP PYQ: Map Coloring

**Question.** A map has four regions: Borogrove, Rath, Brillig, and Tove. Use colors Red, Blue, and Green. Neighboring regions cannot have the same color. Explain how MRV/MCV/LCV help choose the coloring order.

Assume adjacency:

```text
Borogrove adjacent to Rath and Brillig
Rath adjacent to Borogrove and Brillig
Brillig adjacent to Borogrove, Rath, and Tove
Tove adjacent to Brillig
```

### Stepwise Solution

**Step 1: Variables.**

```text
Boro, Rath, Brillig, Tove
```

**Step 2: Domain.**

Each variable has domain:

```text
{Red, Blue, Green}
```

**Step 3: Constraints.**

Adjacent regions must have different colors:

```text
Boro != Rath
Boro != Brillig
Rath != Brillig
Brillig != Tove
```

**Step 4: Choose variable.**

Using MCV, choose Brillig first because it constrains three other regions.

Let:

```text
Brillig = Red
```

**Step 5: Assign neighbors.**

Borogrove and Rath cannot be Red. Let:

```text
Borogrove = Blue
Rath = Green
```

Both are different from Brillig and from each other.

**Step 6: Assign Tove.**

Tove is adjacent only to Brillig, so Tove cannot be Red. Choose:

```text
Tove = Blue
```

### Final Exam Answer

One valid coloring is:

| Region | Color |
|---|---|
| Brillig | Red |
| Borogrove | Blue |
| Rath | Green |
| Tove | Blue |

MRV chooses a variable with minimum legal values, MCV chooses the most constraining variable, and LCV chooses the value that leaves maximum choices for neighboring variables. Here Brillig is a good first variable because it touches the most regions.

## Q10. Alpha-Beta Pruning on Q9

**Question.** For the same tree in Q9, apply alpha-beta pruning from left to right.

### Stepwise Solution

**Step 1: Evaluate M1.**

M1 is MIN:

```text
3, 10, 5, 4
```

Minimum is 3. Root alpha becomes:

$$
\alpha = 3
$$

**Step 2: Evaluate M2.**

M2 is MIN. Values are:

```text
15, 6, 2, 1
```

At M2:

- See 15, beta = 15.
- See 6, beta = 6.
- See 2, beta = 2.

Now:

$$
\beta = 2 \le \alpha = 3
$$

So the remaining child 1 can be pruned.

**Step 3: Evaluate M3.**

M3 values:

```text
6, 2, 8, 5
```

At M3:

- See 6, beta = 6.
- See 2, beta = 2.

Again:

$$
\beta = 2 \le \alpha = 3
$$

So remaining children 8 and 5 are pruned.

**Step 4: Evaluate M4.**

M4 values:

```text
11, 5, 7, 6
```

Minimum is 5. Root chooses max of 3 and 5:

$$
\alpha = 5
$$

### Final Exam Answer

Alpha-beta pruning removes branches that cannot affect the final minimax value. The final minimax value remains:

$$
5
$$

The best move is still:

```text
M4
```

Pruned branches:

```text
M2: last child after beta becomes 2
M3: last two children after beta becomes 2
```

## Q11. Nim Game PYQ Pattern

**Question.** Nim starts with one stack of 7 tokens. At each move, a player splits one stack into two non-empty unequal stacks. A player who cannot move loses. Draw the search tree and decide whether the first player has a winning move.

### Stepwise Solution

**Step 1: Legal splits of 7.**

A stack can be split into unequal positive parts:

```text
7 -> (1,6), (2,5), (3,4)
```

**Step 2: Analyze terminal stacks.**

A stack of 1 cannot be split. A stack of 2 cannot be split into unequal positive parts because only \(1+1\) is possible and equal. Therefore stacks 1 and 2 are terminal.

**Step 3: Analyze moves.**

From 7:

- \(7 \to (1,6)\)
- \(7 \to (2,5)\)
- \(7 \to (3,4)\)

Now analyze whether the next player can force a win.

Useful sub-results:

```text
3 -> (1,2), then no further split in either stack.
4 -> (1,3), then 3 can split to (1,2).
5 -> (1,4), (2,3)
6 -> (1,5), (2,4)
```

**Step 4: Winning idea.**

The first player should move to a position where every move of the opponent gives the first player a reply. For 7-token Nim, a standard good split is:

```text
7 -> (3,4)
```

If opponent splits 3 into (1,2), the remaining 4 can still be split. If opponent splits 4 into (1,3), the remaining 3 can still be split. The first player can force the last move.

### Final Exam Answer

The initial legal moves are:

```text
(1,6), (2,5), (3,4)
```

The first player has a winning strategy by choosing:

```text
7 -> (3,4)
```

This creates two active stacks and allows the first player to respond to the opponent’s split. The complete search tree should show legal unequal splits until only stacks of size 1 or 2 remain.

## Q13. Painter Problem 2: STRIPS Operator for Paint

```text
not Unpainted(object)
```

STRIPS operator:

```text
Action: Paint(object, color)

Object(object)

Painted(object, color)
```

This operator is correct because it clearly states what must be true before painting and how the state changes after painting.

## Q15. POP Threats: Fried Eggs Pattern

**Question.** A partial order planner is making fried eggs. It has causal links involving raw eggs, cooked eggs, and eggs in a dish. A new action threatens two causal links. Explain how threats are identified and resolved.

### Stepwise Solution

**Step 1: Define causal link.**

A causal link:

$$
A \xrightarrow{p} B
$$

means action A establishes condition \(p\), which is needed by action B.

**Step 2: Define threat.**

An action T is a threat if it can occur between A and B and deletes \(p\).

For example:

```text
Put(eggs, pan) -- in(eggs, pan) --> Cook(eggs)
```

If an action:

```text
Put(eggs, dish)
```

deletes:

```text
in(eggs, pan)
```

then it threatens the causal link from `Put(eggs, pan)` to `Cook(eggs)`.

**Step 3: Resolve by promotion.**

Promotion puts the threatening action after the consumer:

```text
Cook(eggs) < Put(eggs, dish)
```

**Step 4: Resolve by demotion.**

Demotion puts the threatening action before the producer:

```text
Put(eggs, dish) < Put(eggs, pan)
```

This may or may not be meaningful depending on preconditions.

### Final Exam Answer

Threats in POP occur when a new action can delete a condition protected by a causal link. They are resolved using:

```text
Promotion: put the threatening action after the consumer.
Demotion: put the threatening action before the producer.
```

For fried eggs, if `Put(eggs,dish)` deletes `in(eggs,pan)`, it threatens the link needed by `Cook(eggs)`. A valid resolution is:

```text
Cook(eggs) < Put(eggs,dish)
```

## Q16. Propositional Translation Set 1

**Question.** Translate into propositional logic:

1. It is raining if and only if Liron is sick.
2. If Liron is sick then it is raining, and vice versa.
3. It is raining is equivalent to Liron is sick.
4. Liron is hungry but happy.

### Stepwise Solution

Let:

```text
R: It is raining
S: Liron is sick
H: Liron is hungry
P: Liron is happy
```

1. "If and only if" means biconditional:

$$
R \leftrightarrow S
$$

2. "If Liron is sick then it is raining, and vice versa" means both implications:

$$
(S \to R) \land (R \to S)
$$

3. "Equivalent" also means biconditional:

$$
R \leftrightarrow S
$$

4. "But" is logically conjunction:

$$
H \land P
$$

### Final Exam Answer

The translations are:

```text
1. R <-> S
2. (S -> R) and (R -> S)
3. R <-> S
4. H and P
```

## Q17. Propositional Translation Set 2

2. You will qualify GATE only if you work hard.
3. Birds fly if and only if the sky is clear.

```text
R: It rains
H: I will stay at home
Q: You qualify GATE
W: You work hard
B: Birds fly
C: Sky is clear
M: He goes to play a match
```

$$
Q \to W
$$

B \leftrightarrow C
$$

$$
M \leftrightarrow \neg R
$$

### Final Exam Answer

```text
1. R -> H
2. Q -> W
3. B <-> C
```

## Q24. Explain Learning Agent with Suitable Example

## Q25. Explain Task Environment Properties

### Exam Answer

The task environment describes the conditions under which an agent operates. Its properties determine the design of the agent.

1. **Fully observable vs partially observable:** If sensors provide complete information, the environment is fully observable. If information is incomplete, it is partially observable.
2. **Deterministic vs stochastic:** If the next state is completely determined by current state and action, it is deterministic. If uncertainty exists, it is stochastic.
3. **Episodic vs sequential:** In episodic environments, each decision is independent. In sequential environments, current actions affect future decisions.
4. **Static vs dynamic:** A static environment does not change while the agent decides. A dynamic environment may change during decision-making.
5. **Discrete vs continuous:** Discrete environments have finite distinct states/actions, while continuous environments have continuous values.
6. **Single-agent vs multi-agent:** A single-agent environment has one agent; a multi-agent environment has multiple interacting agents.

Example: chess is fully observable, strategic, sequential, mostly static, discrete, and multi-agent. Taxi driving is partially observable, stochastic, sequential, dynamic, continuous, and multi-agent.

## Q26. Why BFS and DFS are Called Weak Methods

### Exam Answer

BFS and DFS are called **weak methods** because they do not use domain-specific knowledge or heuristics. They search blindly through the state space.

BFS expands nodes level by level and can find the shortest path when all costs are equal. However, it requires large memory because it stores many nodes.

DFS uses less memory but may go deep into an unproductive branch. It is not complete in infinite-depth spaces and is not optimal.

Both methods are general-purpose and easy to apply, but they become inefficient for large search spaces. Therefore, they are called weak or uninformed methods.

## Q32. Explain PROLOG Backtracking

### Exam Answer

Backtracking in PROLOG is the process of trying alternative facts or rules when the current choice fails. PROLOG searches depth-first. If a subgoal fails, it returns to the most recent choice point and tries another possible match.

Example:

```prolog
likes(raj, tea).
likes(raj, coffee).
likes(sita, tea).

drink(X) :- likes(X, tea).
```

Query:

```prolog
?- drink(X).
```

PROLOG first tries `X=raj` because `likes(raj, tea)` matches. If the user asks for more solutions, PROLOG backtracks and finds `X=sita`.

Thus, backtracking allows PROLOG to explore multiple possible solutions automatically.

## Q34. Limitations of Goal Stack Planning

### Exam Answer

Goal stack planning solves planning problems by pushing goals, subgoals, and actions onto a stack. It is simple and useful for problems like Blocks World.

However, it has limitations:

1. It may commit to subgoal order too early.
2. Achieving one goal may undo another goal.

## Q35. Explain Partial Order Planning

### Exam Answer

Partial Order Planning is a planning method that does not fix the order of actions unless required. It follows the principle of **least commitment**.

A partial plan contains:

1. Actions
2. Ordering constraints
3. Causal links
4. Open preconditions

A causal link:

$$
A \xrightarrow{p} B
$$

means action A achieves condition \(p\), which is needed by action B.

If another action can delete \(p\) between A and B, it is a **threat**. Threats are resolved by:

```text
Promotion: put threat after B
Demotion: put threat before A
```

POP is useful because it keeps plans flexible and avoids unnecessary ordering decisions.

## Q36. Explain A* and Conditions for Optimality

### Exam Answer

A* is an informed search algorithm that uses:

$$
f(n)=g(n)+h(n)
$$

where \(g(n)\) is the cost from the start node to \(n\), and \(h(n)\) is the estimated cost from \(n\) to the goal.

A* expands the node with the smallest \(f(n)\). It combines Uniform Cost Search and Greedy Best First Search.

A* is optimal when the heuristic is **admissible**, meaning:

$$
0 \le h(n) \le h^*(n)
$$

It is especially efficient with a **consistent** heuristic:

$$
h(n) \le c(n,n') + h(n')
$$

Thus, A* finds the least-cost path if the heuristic does not overestimate the true cost.

## Q37. Explain Forward and Backward Chaining

### Exam Answer

Forward chaining and backward chaining are inference methods used mainly with Horn clauses.

**Forward chaining** is data-driven. It starts with known facts and applies rules whose premises are true. It continues until the required conclusion is derived or no new facts can be produced.

Example:

```text
Fact: A
Rule: A -> B
Rule: B -> C
```

Forward chaining derives B and then C.

**Backward chaining** is goal-driven. It starts with the query and tries to prove it by proving its subgoals.

For query C, it checks rule \(B -> C\), then tries to prove B, then checks \(A -> B\), then proves A from facts.

Forward chaining is useful when many conclusions are needed. Backward chaining is useful when a specific query must be proved.

## Q39. Explain GraphPlan

### Exam Answer

GraphPlan is a planning algorithm that uses a planning graph to restrict search.

A planning graph is a leveled graph with:

1. Proposition levels
2. Action levels
3. Precondition edges
4. Add-effect edges
5. Delete-effect edges
6. Mutex relations

The algorithm works as follows:

1. Start with initial propositions.
2. Add all actions whose preconditions are satisfied.
3. Add their effects to the next proposition level.
4. Mark mutually exclusive actions and propositions.
5. Continue until all goals appear without mutex.
6. Search backward for a valid plan.

GraphPlan is sound, complete, and useful for finding shortest parallel plans in STRIPS-style domains.

## Q40. Explain Means-Ends Analysis

### Exam Answer

Means-Ends Analysis is a problem-solving strategy that reduces the difference between the current state and the goal state.

Steps:

1. Compare current state with goal state.
2. Identify the difference.
3. Choose an operator that reduces the difference.
4. If the operator’s preconditions are not satisfied, create subgoals.
5. Apply the operator when preconditions are satisfied.

Example: If the goal is to travel from Pune to Delhi and the current state is being in Pune, the difference is location. Operators may include walking, driving, bus, cab, or flying. If flying is chosen, preconditions like having a ticket and being at the airport become subgoals.

Thus, means-ends analysis solves problems by repeatedly reducing differences between current and goal states.

## Q41. Explain AO*

### Exam Answer

AO* is a heuristic search algorithm used for AND-OR graphs.

In an OR node, one child is selected because alternative solutions are available. In an AND node, all children must be solved because the problem is decomposed into subproblems.

AO* works by:

1. Starting at the root.
2. Expanding the most promising node using heuristic estimates.
3. Updating costs of parent nodes.
4. Marking the best solution subgraph.
5. Continuing until the solution graph is solved.

AO* differs from A* because A* finds a single best path, while AO* finds a best solution subgraph in an AND-OR graph.

## Q42. Explain Waiting for Quiescence

### Exam Answer

Waiting for quiescence is used in game playing. A position is called quiescent when it is stable and the evaluation function is unlikely to change suddenly.

If a game tree is cut off at an unstable position, the evaluation may be misleading. For example, in chess, evaluating a position while a piece is about to be captured may give a wrong estimate.

Therefore, the search is extended until a quiet or stable position is reached. Then the evaluation function is applied.

Thus, waiting for quiescence improves the reliability of game-tree evaluation.
